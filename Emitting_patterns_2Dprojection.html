Emitting Patterns 2D ProjectionÔÉÅ

In this tutorial we show how to use the 2D projection tool.

The function used below needs the file xpsi/utilities/ProjectionTool.py. You can also use this file and the corresponding functions without having X-PSI installed.

X-PSI models have one or two hot spots. According to the X-PSI naming convention we refer to them as the primary (‚Äòp‚Äô) and secondary (‚Äòs‚Äô), if two hot spots are modeled.

The plot_projection_general function takes at least 3 arguments:

  • Input parameters [dict]: in the form of X-PSI likelihood object or dictionary (for examples on use of the former see Modeling tutorial);

  • model [string]: model describing the emission (this information overwrite the information given by the set of passed parameters);

  • Point Of View [string/list/vector]: the projection requires the definition of a point of view. This is given as a string argument or as a 3D vector. The string argument describes points of view associated with the model. One that is common for every model is ‚ÄúI‚Äù, this takes as entry the value associated to ‚Äòcos_incliation‚Äô as colatitude defining the center of the projection; in this case the phase is set to the X-PSI defined 0.0 phase. Other string possibilities depend on the model and could be ‚ÄúPE‚Äù, origin of the projection set to the primary emitting component (when the primary has also an omitting component) or ‚ÄúP‚Äù or ‚ÄúS‚Äù if respectively the primary or the secondary has a single component. If the wrong entry is provided, suggestions are given; in general: the first letter P/S stands for primary/secondary; the second letter S/C stands for super/cede component (if a ceding component is present); E/O stand for emitting/omitting component (if an omitting component is present). It is also possible to give a vector (describing a point of the surface from the origin=center of the unitary sphere) as a point of view; that would direct the center of the projection to that point.

Additional optional arguments are:

  • ThetaDisplay [string]: plots in one hemisphere some circles, at equal distance in colatitude, to give an idea of the colatitude values, ‚ÄúSP‚Äù and ‚ÄúNP‚Äù for plotting them respectively with center ‚ÄúSouth pole‚Äù or ‚ÄúNorth pole‚Äù; if no argument is given, it takes the origin of the projection as reference point for drawing colatitude circles;

  • antiphase [bool]: replies to the question ‚ÄúIs the secondary hot spot defined with anti-phase on (i.e. defined from an origine 0.5 a cycle away from the origin of the primary hot spot)?‚Äù (yes=True; no = False -default)

  • SaveFlag [bool]: replies to the question ‚ÄúDo you want to save the figure?‚Äù (yes=True; no = False - default)

  • dir [string]: name of the directory where to save the image, if SaveFlag = True (default ‚Äú‚Äù)

  • Name [string]: additional string to add to the name of the figure, if SaveFlag = True (default ‚Äú‚Äù)

  • POVname [string]: string specifying the point of view, added to the name of the figure, if SaveFlag = True (default ‚Äú‚Äù)

  • extension [string]: string specifying the extension of the figure (default ‚Äú.png‚Äù)

  • antipodal [bool]: replies to the question ‚ÄúDo you want to also plot a fictitious hot spot antipodal to the primary?‚Äù (yes=True; no = False - default)

Please note that warnings produced with this tool do not cover all the extensive checks and conditions in place within X-PSI

For example there are no checks requiring the overlap of two components of the same hot spot and similarly there is no specific test (and therefore warning) checking if the superseding component is indeed in an ‚Äúeccentric‚Äù location, if so specified by the model name. There are also no tests checking if the superseding or the omitting region completely covers the other component, unless the model is specified to be concentric.

TEST B) 2-hot spot modelsÔÉÅ

In X-PSI we can also define models with 2 hot spots, a primary one and a secondary one. According to the X-PSI naming convention, there are two ways to build a two hot spot model:

  • using two hot spots with different associated names, e.g. ST and PST, that becomes ST+PST model;

  • using the same geometry for both the hot spots, with the possible addition of restrictions on the parameter values of the secondary. We define them adding after the name of the hot spot:

    • -S: this means that all the properties of the secondary hot spot are derived from the properties of the primary hot spot; in particular we assume the secondary to have the same temperature and angular radius of the primary and to be antipodal respect to the primary.

    • -Ua: models ending in this way (as in Salmi et al. 2022) constrain the secondary location to be antipodal to the primary but leave the angular radius and temperature free.

    • -U: models ending in this way have no constraints on the properties of the secondary based on the ones of the primary.

TEST B.2.3) Testing 2 hot spot models composed of the same type of hot spot: uncorrelated case

In this case, the two hot spots of the model have the same name but their properties are completely uncorrelated. This is equivalent to writing the hot spot name twice, joined by ‚Äú+‚Äù (e.g. ‚ÄúST-U‚Äù is equivalent to ‚ÄúST+ST‚Äù). If there are more different derived parameters for the two hot spots, the suggestion is to add the corresponding parameters in the dictionary and effectively convert your model to a ‚Äú-U‚Äù model. In this case the points of view allowed include centers of the primary as well of the secondary components.
